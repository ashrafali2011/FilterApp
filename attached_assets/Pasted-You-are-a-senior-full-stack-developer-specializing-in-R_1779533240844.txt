You are a senior full-stack developer specializing in React, Next.js, and Node.js/Express. Your task is to design and develop a complete, production-ready web application for water filter maintenance and cartridge replacement tracking.

**PROJECT OVERVIEW**

Create a responsive web application (desktop and mobile) that helps users track filter cartridge replacement dates, receive maintenance reminders, and manage multiple filters. The application must support both guest mode (no registration) and optional user authentication, with full backup/restore capabilities and a hidden admin panel for banner management.

The architecture should be designed to later wrap into native Android and iOS applications using Flutter, Capacitor, or WebView-based packaging.

**CORE REQUIREMENTS**

User System:
- Guest Mode: Users access all core features without registration; data stored locally in browser only with a clear warning: "Your data is stored only on this browser/device and will not sync across devices unless you log in."
- Optional Login: Username/password authentication that enables cloud sync; registration must remain optional, never mandatory
- Prepare authentication architecture for future Firebase integration

Filter Management:
- Add filters using pre-built templates (Single-stage, 3-stage, 5-stage, 7-stage) or create fully custom filters
- Each filter contains: name, location, installation date, multiple cartridges/stages, notes, and status

Cartridge Management:
- Replace single cartridge or all cartridges at once
- Each cartridge includes: name, stage number, last replacement date, replacement interval, next replacement date, and status color
- Automatic replacement history tracking with default intervals: Stage 1 → 3 months, Stages 2-3 → 6 months, Stages 4-7 → 12 months

Smart Replacement Schedule:
- System automatically calculates next replacement date, remaining days, overdue state, and reminder schedule

Notification System:
- Send notifications 7, 10, and 20 days before replacement (with custom day options)
- Allow enable/disable globally and per-filter

Backup & Restore:
- Export all data (filters, cartridge history, reminder settings, user preferences, themes, local settings) into a compressed .zip file using JSON structure
- Import/restore backup with integrity validation before importing
- Must work completely offline

Hidden Admin Panel:
- Access via PIN code or password
- Features: add advertisement banners, upload banner images, add clickable URLs, enable/disable banners, manage ad positions
- Banner locations: home screen (2 positions) and filter details page
- Production-ready banner upload form, display components, local storage for ads, enable/disable toggle, and secure admin access flow

**USER INTERFACE**

Design Requirements:
- Modern, clean, professional responsive UI with mobile-first approach
- Dashboard displaying all filters as cards (showing filter name, health status, remaining days, last replacement date, quick actions)
- Status colors: Green (healthy), Yellow (near replacement), Red (needs replacement)
- Dark Mode and Light Mode with custom theme color options
- Arabic + English localization with full RTL support
- Smooth animations and fast performance
- Offline-first support with Service Workers and PWA readiness
- SEO-friendly implementation

Quick action buttons on cards: Add filter, Replace cartridge, View history

**TECHNICAL ARCHITECTURE**

Technology Stack:
- **Frontend**: React with Next.js (for web-first development, SEO, and PWA capabilities)
- **Backend**: Node.js/Express with custom database
- **Local database**: IndexedDB or Hive for client-side storage
- **Offline-first architecture** with Service Workers
- **Architecture Pattern**: Clean Architecture or MVVM with modular, scalable structure

Code Quality Standards:
- Best practices throughout (no tests required, but production-ready code)
- Modular, scalable, maintainable structure with reusable components
- Proper error handling and security considerations
- Ready for deployment

**DELIVERABLES**

1. Complete project architecture and folder structure
2. Database schema with all entities and relationships
3. Responsive UI design for all pages and components
4. Core business logic (filter management, cartridge tracking, notification system)
5. Notification system with scheduling logic
6. Backup & restore implementation with validation
7. Authentication system (optional login/register flow)
8. Hidden admin panel with full implementation
9. Banner ads management system (upload, display, enable/disable)
10. Reusable component library
11. Production-ready, scalable code ready for deployment

**CRITICAL NOTES**

- Registration and login must remain completely optional; all core features must be accessible in guest mode
- Display a clear warning for guest users about data storage limitations
- Backup and restore functionality must work entirely offline
- Design must support future native app packaging without major refactoring
- Admin panel access must be secure and hidden from regular users
- Banner system must be responsive and functional across all screen sizes
- Implement with best practices and solid code quality standards throughout
